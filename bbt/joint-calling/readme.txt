1. cmd.ini是命令行信息的详细参数说明，因为涉及license，其中sentieon镜像信息我不是很清楚
2. variant.*.csv文件是按照maggie之前的要求格式给的参数说明文档
3. joint_call.py是对sentieon的joint caling命令行的包装，本次的命令行信息是对该脚本的调用
4. 注意：鉴于3，需要把joint_call.py打包到sentieon镜像当中
5. 特殊要求：
my_vcfs：该参数可以对应多个输入文件，而且要求客户在输入该文件时，必需满足以下条件：
任务发起者所在机构必需和该输入文件所在的数据集的拥有者一致，因为最终的计算结果将跟据该输入信息判断需要输出的信息。

-------分析流程概览----------------------------------------------------------
my_vcfs: 1.vcf ----------------\
                                |-->[joint-calling]-->[all.vcf]-->[extract info of sample_1]-->target.vcf
other_vcfs: 2.vcf, 3.vcf ------/

--------输入文件-------------------------------
>1.vcf:
chr start ref alt filter info format sample_1
1   101   A   G .............................
1   102   A   C .............................
1   103   T   G .............................

>2.vcf:
chr start ref alt filter info format sample_2
1   101   A   T .............................
1   102   A   C .............................
1   103   T   G .............................

>3.vcf:
chr start ref alt filter info format sample_3
1   101   A   G .............................
1   102   A   T .............................
1   103   T   G .............................

--------输出文件，仅保留sample_1对应的信息-------------------------------
>out.vcf:
chr start ref alt filter info format sample_1
1   101   A   G .............................
1   102   A   T .............................
1   103   T   G .............................
